# GitHub Project Manager

## When to use

Use this skill when organizing GitHub work, creating issue templates, labels, milestones, scripts, or Codex project instructions.

## Goals

- Make work easy for Codex to implement phase by phase.
- Keep issues small and testable.
- Add labels by phase/workstream/priority.
- Add acceptance criteria everywhere.

## Workflow

1. Create labels.
2. Create milestones.
3. Create issue templates.
4. Create project setup docs.
5. Create initial epic issues.
6. Link PR checklist to phase checkboxes.

## Done criteria

- Issue templates exist.
- PR template exists.
- Project setup is documented.
- Issues have checkboxes.
