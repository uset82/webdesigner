# Webview Avatar Designer

## When to use

Use this skill when building UI components in `apps/webview`, avatar layout, motion states, accessibility, or theme integration.

## Goals

- Create a calm, expressive assistant UI.
- Use VS Code theme variables.
- Keep motion subtle.
- Respect reduced motion.
- Prevent UI from blocking editor work.

## Workflow

1. Build UI skeleton.
2. Add avatar stage and fallback.
3. Add state-driven visual changes.
4. Add speech bubble and settings.
5. Test light/dark themes and reduced motion.
6. Document behavior.

## Done criteria

- Webview renders.
- States are visible.
- Reduced motion works.
- UI is theme-aware.
